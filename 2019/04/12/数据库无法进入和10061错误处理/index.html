数据库无法进入和10061错误处理

发表于 2019-04-12 | 更新于 2019-05-10 | 分类于 code

在我休息了一天去公司之后，被告知数据库不能使用了，需要重装。我：？？？
不信邪的我决定尝试一下。
第一个错误：Found option without preceding group in config file D:\mysql\mysql-8.0.13-winx64\my.ini at line 1. 在解决完这个问题之后，又出现了一个新的问题：Can’t connect to MYSQL server on ‘localhost’ (10061)

问题截图如下：

接下来一个一个的解决。

Found option without preceding group in config file D:\mysql\mysql-8.0.13-winx64\my.ini at line 1.

这个问题就是所示位置的配置文件的问题，只需要把my.ini用记事本打开，另存为的时候把编码方式修改为 ANS

10061错误

这个错误是Mysql登陆失败的报错，引起的原因有很多，这里只回答我的问题：是因为没有启动MYSQL服务。10061错误完整分析
知道原因就直接打开服务就好，但是在输入net start mysql的时候又出错了。

这个问题的解决办法就是用管理员的身份运行cmd，再启动MYSQL服务。

  • 一次性方法：在系统盘中找到Windows/System32/cmd.exe右击选择以管理员身份运行。
  • 永久性方法：把cmd.exe的快捷方式放到桌面，右击选择 属性 ->快捷方式 ->高级 ->以管理员身份运行 ->确定。 以后点击快捷方式就是以管理员运行了。


之后就可以正常运行了。很简单，没有想象中那么大的问题。（同事说的时候还吓了我一跳）
